**In `includes/class-cpt-project.php`, add two new static methods:**

1. **`validate_rest_github_url`** — A `rest_pre_insert_community_project` filter callback. This filter fires before a community_project is inserted or updated via REST. Check if `$request['meta']['_community_master_github_url']` is set and non-empty. If so, validate it starts with `https://github.com/`. If it does not, return a `WP_Error` with code `rest_invalid_github_url`, message `'GitHub URL must start with https://github.com/'`, and status 400. If valid or not set, return `$post` unchanged. Method signature: `public static function validate_rest_github_url( $post, WP_REST_Request $request )`.

2. **`register_rest_fields`** — A `rest_api_init` action callback. Uses `register_rest_field('community_project', 'menu_order', [...])` to expose the `menu_order` field in REST responses and allow writing it. The `get_callback` reads `$post['id']` and returns `(int) get_post($post['id'])->menu_order`. The `update_callback` receives `$value, $post` and calls `wp_update_post(['ID' => $post->ID, 'menu_order' => (int) $value])`. The `schema` should be `['type' => 'integer', 'description' => 'Display order for project tiles', 'context' => ['view', 'edit']]`.

**In `includes/class-community-master.php`, wire the new hooks in the constructor:**

Add after the existing `add_action('init', ...)` lines:
```php
add_filter('rest_pre_insert_community_project', [CM_CPT_Project::class, 'validate_rest_github_url'], 10, 2);
add_action('rest_api_init', [CM_CPT_Project::class, 'register_rest_fields']);
```

Do NOT create a custom WP_REST_Controller class. The built-in CPT REST endpoints are sufficient. We are only adding a validation filter and a custom REST field.

Create `tests/test-rest-api.sh` — a bash script that verifies all REST API operations using curl. The script takes two arguments: `$1` = WordPress site URL (e.g., `https://meintechblog.de`), `$2` = Application Password credentials in `user:password` format.

The script should:

1. **Header:** `#!/usr/bin/env bash` with `set -euo pipefail`. Print usage if fewer than 2 args.

2. **Variables:**
   - `BASE_URL="${1%/}/wp-json/wp/v2/community_project"`
   - `AUTH="$2"`
   - `PASS=0` and `FAIL=0` counters
   - A helper function `assert_status()` that takes a test name, expected HTTP status, and the actual curl response, compares them, prints PASS/FAIL, increments counters.

3. **Test: Unauthenticated POST returns 401 (SEC-05)**
   ```bash
   STATUS=$(curl -s -o /dev/null -w "%{http_code}" -X POST "$BASE_URL" \
     -H "Content-Type: application/json" \
     -d '{"title":"Unauth Test","status":"publish"}')
   assert_status "Unauthenticated POST rejected" "401" "$STATUS"
   ```

4. **Test: Create project with all meta fields (API-01, API-04)**

   ```json
     "title": "REST Test Project",
     "status": "publish",

       "_community_master_description": "A test project created via REST API",
       "_community_master_github_url": "https://github.com/meintechblog/test-project",
       "_community_master_installer": "curl -sSL https://example.com/install.sh | bash"
     },
     "menu_order": 5
   }
   ```
