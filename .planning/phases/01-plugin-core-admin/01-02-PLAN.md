Create `includes/class-meta-boxes.php` with class `CM_Meta_Boxes`:

**Constructor** (self-registering hooks):
- `add_action('add_meta_boxes', [$this, 'register_meta_boxes'])`
- `add_action('save_post_community_project', [$this, 'save_meta'], 10, 2)`

**Method `register_meta_boxes(): void`:**
- Register meta box `community_master_fields` with title "Project Details" on screen `community_project`, context `normal`, priority `high`. Render callback: `[$this, 'render_fields_meta_box']`
- Register meta box `community_master_sort_order` with title "Sortierung" on screen `community_project`, context `side`, priority `default`. Render callback: `[$this, 'render_sort_order_meta_box']`

**Method `render_fields_meta_box(WP_Post $post): void`:**
- Output nonce field: `wp_nonce_field('community_master_save_meta', 'community_master_nonce')` (per SEC-04)
- Get current values: `get_post_meta($post->ID, '_community_master_description', true)`, same for `_community_master_github_url` and `_community_master_installer`
- Render a `<table class="form-table">` with three rows:
  - Row 1: Label "Beschreibung" with `<textarea>` named `_community_master_description`, 5 rows, full width, cols=50. Value: `esc_textarea($description)`
  - Row 2: Label "GitHub URL" with `<input type="url">` named `_community_master_github_url`, full width, placeholder `https://github.com/org/repo`. Value: `esc_attr($github_url)`
  - Row 3: Label "One-Line-Installer" with `<input type="text">` named `_community_master_installer`, full width, placeholder `curl -sSL https://example.com/install.sh | bash`. Value: `esc_attr($installer)`

**Method `render_sort_order_meta_box(WP_Post $post): void`:**
- Render `<label>` with text "Reihenfolge (niedrigere Zahl = weiter oben)"
- Render `<input type="number" name="community_master_menu_order" min="0" step="1">` with value `esc_attr($post->menu_order)`, style `width: 100%`
- This uses `menu_order` from wp_posts directly, NOT a custom meta field (per FIELD-06, per research open question 1 recommendation: custom meta box instead of page-attributes to avoid unwanted Parent dropdown)

**Method `save_meta(int $post_id, WP_Post $post): void`:**
Follow the exact save pattern from research (Pattern 3), in this order:

1. Verify nonce: `$nonce = isset($_POST['community_master_nonce']) ? sanitize_text_field(wp_unslash($_POST['community_master_nonce'])) : '';` then `if (!wp_verify_nonce($nonce, 'community_master_save_meta')) { return; }` (per SEC-04)
2. Check autosave: `if (defined('DOING_AUTOSAVE') && DOING_AUTOSAVE) { return; }`
3. Check permissions: `if (!current_user_can('edit_community_project', $post_id)) { return; }`
4. Sanitize and save description: `sanitize_textarea_field(wp_unslash($_POST['_community_master_description']))` then `update_post_meta(...)` (per SEC-01, research pitfall 4: always wp_unslash before sanitize)
5. Sanitize and save GitHub URL: `esc_url_raw(wp_unslash($_POST['_community_master_github_url']))` then validate prefix `https://github.com/` -- if URL is non-empty and does not start with `https://github.com/`, set to empty string. Then `update_post_meta(...)` (per FIELD-04 validation)
6. Sanitize and save installer: `sanitize_text_field(wp_unslash($_POST['_community_master_installer']))` then `update_post_meta(...)`
7. Save menu_order: If `$_POST['community_master_menu_order']` is set, cast to `(int)` and call `wp_update_post(['ID' => $post_id, 'menu_order' => $order])` -- unhook save_post first to prevent infinite loop: `remove_action('save_post_community_project', [$this, 'save_meta'])`, then `wp_update_post(...)`, then `add_action('save_post_community_project', [$this, 'save_meta'], 10, 2)` (per FIELD-06)

Also update `includes/class-community-master.php` to instantiate `CM_Meta_Boxes`:
- In the private constructor, add: `new CM_Meta_Boxes();`

Create `includes/class-admin-columns.php` with class `CM_Admin_Columns` (per D-03):

**Constructor** (self-registering hooks):
- `add_filter('manage_community_project_posts_columns', [$this, 'set_columns'])`
- `add_action('manage_community_project_posts_custom_column', [$this, 'render_column'], 10, 2)`
- `add_filter('manage_edit-community_project_sortable_columns', [$this, 'set_sortable_columns'])`

**Method `set_columns(array $columns): array`:**
- Build new column array by iterating existing `$columns`. After the `'title'` key, insert:
  - `'cm_logo' => __('Logo', 'community-master')`
  - `'cm_github' => __('GitHub URL', 'community-master')`
  - `'cm_sort' => __('Sortierung', 'community-master')`
- Return the rebuilt array (preserving cb checkbox and title, inserting custom columns after title)

**Method `render_column(string $column, int $post_id): void`:**
- Switch on `$column`:
  - Case `'cm_logo'`: If `has_post_thumbnail($post_id)`, echo `get_the_post_thumbnail($post_id, [40, 40])`. Else echo `'&mdash;'`.
  - Case `'cm_github'`: Get `get_post_meta($post_id, '_community_master_github_url', true)`. If non-empty, echo `'<a href="' . esc_url($url) . '" target="_blank" rel="noopener noreferrer">' . esc_html($url) . '</a>'`. Else echo `'&mdash;'`.
  - Case `'cm_sort'`: Get `get_post($post_id)`, echo `esc_html((string) $post->menu_order)`.

**Method `set_sortable_columns(array $columns): array`:**
- Add `$columns['cm_sort'] = 'menu_order';`
- Return `$columns`

Also update `includes/class-community-master.php` to instantiate `CM_Admin_Columns`:
- In the private constructor, add: `new CM_Admin_Columns();`
