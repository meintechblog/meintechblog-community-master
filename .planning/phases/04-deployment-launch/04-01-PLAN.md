Per D-04 and D-06, push the plugin code to GitHub via SSH and create a release with a distributable ZIP.

**Step 1 — Create GitHub repo (if it does not already exist):**
```bash
gh repo create meintechblog/meintechblog-community-master --public --description "WordPress plugin: Community project showcase for meintechblog.de" 2>/dev/null || echo "Repo may already exist"
```
If `gh` auth fails, output a message asking the user to create the repo manually at https://github.com/new and resume. Do NOT block — proceed to Step 2 assuming the repo exists.

**Step 2 — Set SSH remote and push:**
```bash
git remote set-url origin git@github.com:meintechblog/meintechblog-community-master.git 2>/dev/null || git remote add origin git@github.com:meintechblog/meintechblog-community-master.git
git push -u origin main
```

**Step 3 — Build distributable ZIP (per D-01):**
Create the ZIP from the project root, including ONLY these paths:
```bash
cd /Users/hulki/codex/meintechblog-community-master
zip -r community-master.zip \
  community-master.php \
  uninstall.php \
  includes/ \
  templates/ \
  assets/ \
  -x "*.DS_Store"
```
The ZIP must NOT contain: `.planning/`, `tests/`, `.git/`, `node_modules/`, `CLAUDE.md`.

**Step 4 — Create GitHub release v1.0.0 with the ZIP:**
```bash
gh release create v1.0.0 community-master.zip \
  --title "v1.0.0 — Initial Release" \
  --notes "Community Master WordPress plugin — initial release. Install via WordPress Plugin Upload (Plugins > Add New > Upload)."
```
If `gh` is not authenticated, output the manual steps: go to GitHub repo > Releases > Create release > tag v1.0.0 > upload community-master.zip.

  <action>
Per D-02 and D-06, install the plugin on the production WordPress site.

**Strategy:** WordPress REST API does not have a native "install plugin from ZIP" endpoint. Use the WordPress Application Passwords approach to verify the site is reachable, then attempt installation via the plugin install endpoint if available, otherwise guide the user.

**Step 1 — Verify WordPress REST API is reachable and authenticated:**
```bash
curl -s -o /dev/null -w "%{http_code}" \
  -u "hulki:nfKk err4 PHmj OI5M Bda2 NHJz" \
  "https://meintechblog.de/wp-json/wp/v2/users/me"
```
Expected: HTTP 200. If not 200, report the error and stop.

**Step 2 — Check if wp-cli is available via SSH (preferred method):**
Try to detect if wp-cli can be used. If SSH access to meintechblog.de is available:
```bash
ssh meintechblog.de "which wp" 2>/dev/null
```
If wp-cli is available:
```bash
scp community-master.zip meintechblog.de:/tmp/
ssh meintechblog.de "cd /var/www/html && wp plugin install /tmp/community-master.zip --activate --force"
```

**Step 3 — Fallback: WordPress plugin upload via REST API (WP 5.5+):**
WordPress 5.5+ supports plugin installation via REST API:
```bash
curl -s -w "\n%{http_code}" \
  -u "hulki:nfKk err4 PHmj OI5M Bda2 NHJz" \
  -X POST "https://meintechblog.de/wp-json/wp/v2/plugins" \
  -F "file=@community-master.zip" \
  -H "Content-Disposition: attachment; filename=community-master.zip"
```
Note: This endpoint may or may not be available. If it returns 404 or fails, proceed to Step 4.

**Step 4 — Final fallback: Manual install checkpoint:**
If neither wp-cli nor REST plugin install works, output clear instructions:
1. Go to https://meintechblog.de/wp-admin/plugin-install.php?tab=upload
2. Choose the `community-master.zip` file (download from GitHub release: https://github.com/meintechblog/meintechblog-community-master/releases/download/v1.0.0/community-master.zip)
3. Click "Install Now", then "Activate Plugin"

**Step 5 — Verify plugin is active:**
```bash
curl -s -u "hulki:nfKk err4 PHmj OI5M Bda2 NHJz" \
  "https://meintechblog.de/wp-json/wp/v2/community_project" \
  -o /dev/null -w "%{http_code}"
```
Expected: HTTP 200 (the CPT endpoint exists, meaning the plugin is active). If 404, the plugin is not active yet.
